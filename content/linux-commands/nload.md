
# 📶 `nload`: кратко и по делу
`nload` — интерактивная утилита для **графического мониторинга сетевого трафика** в реальном времени.  
Показывает входящий и исходящий трафик, среднюю и пиковую скорость, а также общий объём переданных данных.

---

## 📝 Топ‑20 ключевых флагов и приёмов `nload`
- `nload` — запустить мониторинг всех интерфейсов.
- `nload eth0` — указать конкретный интерфейс.
- `nload -m` — показать все интерфейсы в одном окне.
- `nload -u M` — показывать скорость в мегабайтах.
- `nload -u m` — скорость в мегабитах.
- `nload -u K` — скорость в килобайтах.
- `nload -U k` — суммарный трафик в килобайтах.
- `nload -t 500` — обновлять статистику каждые 500 мс.
- `nload -a 500` — устанавливать интервал усреднения (мс).
- `nload -i 1024` — лимитировать верхнюю шкалу входящего графика.
- `nload -o 512` — лимитировать верхнюю шкалу исходящего графика.
- `nload -s 10` — ширина графика в символах.
- `nload -f /tmp/nload.log` — логировать данные в файл.
- `nload -p` — показывать проценты использования пропускной способности.
- `nload -U M` — объём данных в мегабайтах.
- `nload -h` — справка.
- `nload -V` — версия.
- `nload | grep eth0` — фильтрация вывода по интерфейсу.
- `nload -m -u M -U M` — полный мониторинг в мегабайтах.
- `sudo nload -m -u m -U m -t 500` — оптимальный режим мониторинга.

# 🚀 Айсберг команд `nload` — от базового к гуру

## 🟢 Уровень 1 — База (разогрев)

`nload`
- Мониторинг всех интерфейсов

`nload eth0`
- Показать только интерфейс eth0

`nload wlan0`
- Мониторинг Wi-Fi интерфейса

`nload -m`
- Все интерфейсы в одном экране

`nload -u M`
- Скорость в мегабайтах


## 🟡 Уровень 2 — Настройка отображения

`nload -u m`
- Скорость в мегабитах

`nload -u K`
- Скорость в килобайтах

`nload -U M`
- Объём данных в мегабайтах

`nload -U k`
- Объём данных в килобайтах

`nload -p`
- Включить отображение процентов нагрузки


## 🟠 Уровень 3 — Управление шкалами графика

`nload -i 2048`
- Верхняя шкала входящего графика = 2048 KB/s

`nload -o 1024`
- Верхняя шкала исходящего графика = 1024 KB/s

`nload -s 20`
- Ширина графика 20 символов

`nload -a 1000`
- Интервал усреднения 1 секунда

`nload -t 250`
- Интервал обновления 250 мс


## 🔵 Уровень 4 — Логирование и автоматизация

`nload -f /tmp/net.log`
- Сохранять статистику в файл

`nload eth0 -f /tmp/eth0.log`
- Лог только по одному интерфейсу

`cat /tmp/net.log`
- Просмотр сохранённых данных

`tail -f /tmp/net.log`
- Реальное время из лога

`nload -m -f /var/log/nload.log`
- Логировать все интерфейсы


## 🟣 Уровень 5 — Сравнение интерфейсов

`nload -m`
- Все интерфейсы в одном экране

`nload -m -u M`
- Все интерфейсы в мегабайтах

`nload -m -u m -U m`
- Входящий/исходящий в мегабитах

`nload -m -t 200`
- Обновление каждые 200 мс

`nload -m -p`
- С процентами использования


## 🔴 Уровень 6 — Гуру‑режим

`sudo nload -m -u M -U M -t 100`
- Максимальная детализация в мегабайтах

`sudo nload -i 4096 -o 4096`
- Лимит графиков на 4 МБ/c

`sudo nload -m -s 30`
- График шириной 30 символов

`sudo nload eth0 -u m -U m`
- Мониторинг конкретного интерфейса

`sudo nload -m -p -t 200`
- С процентами и быстрым обновлением


## 🟤 Уровень 7 — Интеграция и автоматизация

`watch -n1 nload -m`
- Автообновление каждые 1 секунду

`nload -m | tee nload.log`
- Сохранять вывод в файл

`grep "eth0" nload.log`
- Найти статистику по конкретному интерфейсу

`awk '{print $1,$2}' nload.log`
- Парсинг логов для анализа

`gnuplot nload.log`
- Построение графика по данным


## 🧩 Бонус — практические сценарии

**Мониторинг Ethernet-интерфейса:**
```
nload eth0
```

**Следить за всеми интерфейсами сразу:**
```
nload -m
```

**Показывать скорость в мегабитах:**
```
nload -u m
```

**Сохранение статистики в лог:**
```
nload -f /tmp/nload.log
```

**Мониторинг Wi-Fi в реальном времени:**
```
nload wlan0 -u M -U M
```
