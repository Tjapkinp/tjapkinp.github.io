`pwd` (**p**rint **w**orking **d**irectory) — показывает полный путь к текущей рабочей директории.  
Простая команда, но с полезными флагами и скрытыми трюками для автоматизации и скриптов.

## 📝 Топ‑20 самых полезных флагов и приёмов `pwd`
- `-L` — путь с учётом **символических ссылок** (логический путь, default).
- `-P` — путь, разворачивая все **символические ссылки** (физический путь).
- `--help` — справка по команде.
- `--version` — версия утилиты (`GNU coreutils`).
- `pwd -P | xargs ls -la` — посмотреть содержимое «физической» папки.
- `pwd -L | xargs ls -la` — посмотреть содержимое «логической» папки.
- `pwd | xclip -selection clipboard` — скопировать текущий путь в буфер.
- `pwd | pbcopy` — аналогично на macOS.
- `realpath .` — альтернатива `pwd -P`, но надёжнее для скриптов.
- `cd $(pwd)` — обновить окружение, полезно при сломанных ссылках.
- `pwd | tee current_path.txt` — сохранить текущий путь в файл.
- `pwd && cd /tmp && pwd && cd -` — быстро проверить `cd -` возврат.
- `dirs -v` — список всех директорий в стеке оболочки.
- `pushd $(pwd)` — сохранить текущий путь в стек (`popd` для возврата).
- `echo $PWD` — использовать переменную окружения вместо вызова `pwd`.
- `cd -P . && pwd` — перейти в физический путь без симлинков.
- `readlink -f .` — ещё один способ получить канонический путь.
- `pwd | awk -F/ '{print $NF}'` — вывести только имя текущей директории.
- `basename "$PWD"` — надёжный способ получить только текущую папку.
- `cd $(dirname $(pwd))` — перейти на уровень выше, используя `pwd`.

# 🚀 Айсберг команд `pwd` — от базового к гуру

## 🟢 Уровень 1 — База (разогрев)

`pwd`
- Показать полный путь к текущей директории

`echo $PWD`
- То же самое, но через переменную окружения

`pwd -L`
- Логический путь (по умолчанию)

`pwd -P`
- Физический путь (без симлинков)


## 🟡 Уровень 2 — Работа с симлинками

`ln -s /var/www/html mylink`
- Создать симлинк

`cd mylink && pwd -L`
- Покажет путь **с симлинком**

`cd mylink && pwd -P`
- Покажет реальный физический путь

`realpath mylink`
- Альтернатива `pwd -P`, но более гибкая


## 🟠 Уровень 3 — Скрипты и автоматизация

`pwd | xclip -selection clipboard`
- Скопировать путь в буфер (Linux, X11)

`pwd | pbcopy`
- macOS: скопировать путь

`cd $(pwd)`
- Принудительно обновить текущую директорию (редко, но полезно в bash/zsh)

`pwd | tee /tmp/current_dir.txt`
- Сохраняет путь в файл и выводит на экран

`echo "Текущий путь: $(pwd)"`
- Встраивание `pwd` в скрипт


## 🔵 Уровень 4 — Комбинирование с другими командами

`ls -la $(pwd)`
- Содержимое текущей директории

`tar -czvf "$(basename $(pwd)).tar.gz" .`
- Упаковать текущую директорию в архив, используя её имя

`find $(pwd) -type f -name "*.log"`
- Найти все логи в текущей директории

`git -C $(pwd) status`
- Выполнить git-команду, находясь в текущей директории

`rsync -av $(pwd)/ user@host:/backup/`
- Передать содержимое текущей директории на сервер


## 🟣 Уровень 5 — Фильтрация и парсинг пути

`pwd | awk -F/ '{print $NF}'`
- Только имя текущей директории

`basename "$PWD"`
- Надёжно получить только имя текущей папки

`dirname "$(pwd)"`
- Получить родительскую директорию

`pwd | cut -d/ -f2`
- Вывести часть пути по индексу

`IFS='/' read -ra parts <<< "$(pwd)"; echo ${parts[2]}`
- Разбить путь на массив и взять нужный сегмент


## 🔴 Уровень 6 — Продвинутое использование

`cd -P . && pwd`
- Принудительный переход в физическую директорию

`pwd | sed 's|^/home/|~/|'`
- Заменить начало пути на `~` для компактности

`pwd | tr '/' '\n'`
- Разбить путь на строки по разделителю `/`

`PS1='[\u@\h \W]> '`
- Отображать только имя текущей директории в prompt

`export PROMPT_DIRTRIM=2`
- Сократить путь в bash prompt до 2 уровней


## 🟤 Уровень 7 — Гуру‑режим

`readlink -f .`
- Канонический путь, включая все симлинки

`python3 -c "import os; print(os.getcwd())"`
- Получить текущую директорию из Python

`perl -e 'use Cwd; print getcwd()'`
- Альтернатива через Perl

`strace pwd`
- Посмотреть системные вызовы `pwd` (syscall `getcwd()`)

`cd $(dirname $(readlink -f $(which pwd))) && ./pwd`
- Запуск бинарника `pwd` напрямую, минуя PATH

`pwd && echo "OK" || echo "FAIL"`
- Использовать `pwd` как часть проверки скрипта


## 🧩 Бонус — практические сценарии

**Копировать путь в буфер обмена:**
```
pwd | xclip -selection clipboard
```

**Встроить текущий путь в приглашение bash:**
```
export PS1="[\u@\h \W]$ "
```

**Создать архив текущей директории с её именем:**
```
tar -czvf "$(basename $(pwd)).tar.gz" .
```

**Передать текущую директорию на сервер:**
```
rsync -av $(pwd)/ user@host:/backup/
```
